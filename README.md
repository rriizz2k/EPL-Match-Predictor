# EPL Match Outcome Predictor

Модель машинного обучения для предсказания исхода матчей Английской Премьер-лиги (Победа хозяев / Ничья / Победа гостей) на основе исторической статистики команд.

## Результаты

| Метрика | Значение |
|---|---|
| Accuracy | **53.5%** |
| Бейзлайн (наивная стратегия "всегда хозяева") | 46.5% |
| Алгоритм | Random Forest Classifier |
| Train/Test split | 70/30, хронологический |
| Сезон | 2023-24 (Premier League) |

Модель превосходит наивную стратегию на ~7 процентных пунктов, что подтверждает наличие предсказательного сигнала в построенных фичах.

### Confusion Matrix

| | Предсказано: H | Предсказано: D | Предсказано: A |
|---|---|---|---|
| **Реально: H** | 34 | 5 | 7 |
| **Реально: D** | 13 | 2 | 9 |
| **Реально: A** | 10 | 5 | 14 |

**Известное ограничение:** модель плохо предсказывает ничьи (D) — recall по этому классу крайне низкий (2 из 24). Это распространённая и документированная сложность в задачах прогноза спортивных исходов: ничья статистически "размыта" между двумя явными победами и не имеет чёткого сигнала в данных о форме команд.

## Признаки (features)

Все признаки рассчитаны по принципу скользящего окна (rolling window) на последних 5 матчах каждой команды, без утечки данных из будущего (`.shift(1)` после `.rolling()`):

- `Form_Home`, `Form_Away` — очки, набранные командой за последние 5 матчей
- `Scored_5g_Home`, `Scored_5g_Away` — голы, забитые за последние 5 матчей
- `Conceded_5g_Home`, `Conceded_5g_Away` — голы, пропущенные за последние 5 матчей
- `Form_Diff`, `Scored_Diff`, `Conceded_Diff` — разница соответствующих метрик между хозяевами и гостями

## Источник данных

[football-data.co.uk](https://www.football-data.co.uk) — исторические результаты матчей Premier League, сезон 2023-24.

## Проверенная и отвергнутая гипотеза: данные за несколько сезонов

**Гипотеза:** добавление сезона 2022-23 увеличит обучающую выборку и улучшит предсказание ничьих (класс с наименьшим количеством примеров).

**Реализация:** объединение двух сезонов с весовым коэффициентом (`sample_weight`) — старому сезону присваивался меньший вес (0.5) относительно нового (1.0), чтобы частично компенсировать изменения в составах команд (трансферы, смена тренеров) между сезонами.

**Результат:** гипотеза не подтвердилась. Accuracy упала с 53.5% (один сезон) до 46-48% (два сезона, как с весами, так и без них). Вероятная причина — различия в составах команд между сезонами создают больше шума, чем пользы от увеличения объёма выборки.

**Вывод:** для данной задачи качество и однородность данных оказались важнее их объёма. Эксперимент сохранён в отдельном ноутбуке (`epl_two_seasons_experiment.ipynb`) для прозрачности процесса.

## Как запустить

### Требования

```
pandas
scikit-learn
```

Установка:

```bash
pip install pandas scikit-learn
```

### Запуск

Открыть `epl_prediction.ipynb` в Jupyter Notebook или Google Colab и выполнить ячейки последовательно. Данные загружаются автоматически по ссылке в первой ячейке — отдельная загрузка файлов не требуется.
